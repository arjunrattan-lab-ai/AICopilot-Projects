# Prior Art Scan — Solution Exploration
_Captured: 2026-03-28_

## Key Prior Art Found

### 1. Combined VLM + Scene Classification Architecture (Slack #anno-bypass-confidencebased, Feb 2026)
- **Rakesh Prasanth** proposed running scene classification + VLM in conjunction, reusing AI Collision Annotator infra
- **Shelender Kumar** reported combined VLM+SC for SBV: 96% precision, 86% recall, 61% bypass
- **Achin Gupta** agreed on combined model but wanted to ship CBB (what's available now) while building combined engine by end of Q2
- **Wajahat Kazmi**: SC model integrates into EFS as part of CBB; "not a big lift"
- **Nihar Gupta**: "Objective is to move quickly"
- Architecture decision: reuse collisions AI annotator infra for sequential AI+VLM flows
- VLM alone for SBV: 54% bypass at ~97% precision (Neeraj Dhandia experiments)
- VLM catches contexts SC model misses: airports, non-road environments

### 2. EVE Phase 2 PRD Architecture (Arjun Rattan, active)
- Two parallel paths: CBB (EFS confidence thresholds) and EVE pipeline (Graph service → foundation models → VLMs)
- Graph service decomposition (AI-261): lib-py-graph core + dedicated aiannotation microservice
- Foundation models: DF FM (AI-205), RF FM (AI-205), Collision FM (AI-76)
- DF FM covers: SBV, Cell Phone, Distraction, Yawning, Cam Obstruction, Smoking, Eating
- RF FM covers: Close Following, SSV, Unsafe Lane Change, Unsafe Passing
- SRFM accuracy ~0.92 vs URM baseline ~0.72
- Collision FM: NCM v1 on preview, >90% recall, ~75% bypass

### 3. Safety H1 2026 Planning (Fahad Javed, org-level)
- H1 annotation target: bypass 35%+ of AI events at 99% precision
- Aggressive HC targets require aggressive AI bypassing
- Collision latency targets: p50 <3:00, p80 ~4:30, p90 ~5:30
- AI event latency: ~10 min target, varies by type
- Ranked #1 project: AI Pipeline Revamp & Reliability (Dhiraj Bathija)
- Ranked #2: Annotations Automation / CBB expansion

### 4. Competitive Context (Safety H1 Planning)
- Samsara gaps: side/rear camera, VLM event summaries, AI Box, confidential safety events
- Netradyne gap: Positive Driving, multi-camera
- Motive expects to "differentiate on precision once the availability gap is closed"
- Competitors not specifically named for annotation automation, but implied all are moving toward less human-in-loop

### 5. Existing Architecture Components
- EFS: Event Filtering Service — confidence scoring, threshold checks, bypass routing
- K2Web: persists annotated events to Safety DB
- Graph service: orchestrates foundation model inference
- Statsig: feature gates and dynamic configs for per-behavior/segment thresholds
- Annotation Tool: human review queue with priority tiers
- Two bypass types: K2Web-level (logic-based) and Annotation-level (config-based)
