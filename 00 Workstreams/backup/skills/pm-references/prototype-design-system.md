# Prototype Design System

Visual language for PM prototypes generated by `/atpm-prototype`. Based on the pm-prototype-design-skill.

## Layout

- **Split panel:** LHS interactive mock (50% viewport) + RHS contextual rationale (50%)
- **Navigation:** Bottom bar with prev/next buttons, dot indicators, step title. Arrow key support.
- **Steps:** Each step pairs one design state with its justification.

## Color Palette

| Role | Color | Hex |
|------|-------|-----|
| Background (page) | Cream | `#FAF8F5` |
| Background (panels) | Warm white | `#FFFFFF` |
| Text (primary) | Near black | `#1A1A1A` |
| Text (secondary) | Warm gray | `#6B6B6B` |
| Accent | Motive blue | `#2563EB` |
| RHS: Design direction | Blue | `bg-blue-50 / border-blue-200` |
| RHS: Data stats | Amber | `bg-amber-50 / border-amber-200` |
| RHS: User voice | Teal | `bg-teal-50 / border-teal-200` |
| RHS: Decision callout | Rose | `bg-rose-50 / border-rose-200` |

## Typography

| Element | Font | Weight | Size |
|---------|------|--------|------|
| Headings | Newsreader | 500 | 1.25-1.5rem |
| Body | Karla | 400 | 0.9-1rem |
| Labels/tags | Karla | 700 | 0.75rem |

Fonts loaded via Google Fonts link. Prototypes require network access for font rendering (system fonts used as fallback).

## RHS Section Types

Each rationale section has a colored left border and icon:

- **Design Direction** (blue): Why this design choice was made
- **Data** (amber): Quantitative evidence supporting the decision
- **User Voice** (teal): Customer quotes and qualitative evidence
- **Decision** (rose): Key decision points and trade-offs

## HTML Structure

Single self-contained HTML file (CSS and JS inline). External dependency: Google Fonts for Newsreader and Karla (degrades gracefully to system fonts offline).
