PE → AE → State Machine — How an SSV Event is Born

Runs the ML model (UDM) on every camera frame. Its job is purely object detection — finds stop signs, vehicles, lane markers in the road-facing video stream. Outputs a score + bounding box per frame. Does not make any event decision.

Key configs: ocr_pipeline_enabled, driver_monitor_ai_mirror, pe_logging_enabled

↓

Consumes PE output + vehicle telemetry (speed). Applies thresholds + tolerance windows over time to decide whether a real event occurred. For SSV it asks: "Was the vehicle above the speed threshold for enough consecutive frames while crossing a stop sign?"

AE version is set via Hubble Config (e.g., AE v12). Threshold + windows are configurable per feature.

↓

The state machine is the decision logic for road-facing features — SSV, Close Following, FCW. It uses speed as a key input to gate events. Which speed it uses is controlled by rf_speed_mode.

The Bug + The Fixes

On Class 2 / light-duty OBDII vehicles, the CAN speed signal arrives at the camera already lagged. So at a stop sign, the camera receives speed still reading >3 mph even after the vehicle has fully stopped → state machine fires SSV → false in-cab alert. The ML model (PE) is not broken. The state machine logic is not broken. The input data is bad.

GPS speed on AIDC+ has minimal lag and good accuracy from windshield mount. Switching eliminates all speed-lag-driven false alerts with no change to speed threshold settings (3 mph trial stays as-is). Backtested on Close Following and ULC — no regression.

Replaces the static config with a dynamic anomaly detection + switching algorithm built into firmware. Four detectors monitor the OBD speed signal in real time:

  • Stale value — speed hasn't changed for too long (sensor freeze)
  • False zero — OBD reports 0 mph while camera sees motion
  • Garbage value — implausible instantaneous speed jumps
  • Negative value — negative OBD readings

When any detector fires, the algorithm drops OBD and promotes GPS automatically — per device, per event window. No CHANGE ticket needed. Owner: Moudood Fiaz (AI Firmware). PR #76094 open.
